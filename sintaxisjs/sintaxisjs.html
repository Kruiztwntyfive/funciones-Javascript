La ubicación de código en JavaScript en un archivo HTML es fundamental para el rendimiento y la funcionalidad de una página web. Para ello hay diferentes formas de vincular este código:

Etiqueta script

Se utiliza para incrustar código ejecutable o datos JavaScript en HTML.

Ubicación de la etiqueta script

El script debe ubicarse al final del body del documento, justo antes de la etiqueta de cierre del mismo, para asegurar que el DOM esté completamente cargado antes de que se ejecute el código JavaScript.

Método 1: Vincular un archivo Javascript utilizando el elemento HEAD

En el HTML, dentro de la etiqueta HEAD. Emplearíamos el atributo src para indicar el origen del fichero JavaScript.

Con esto, conseguimos que el navegador cargue y ejecute el script antes de cargar el contenido del body.

Normalente este método se utiliza para cargar funciones que no dependen del contenido del body o para enlazar archivos de forma asíncrona/diferida.

Método 2: Añade tu código Javascript en el encabezado de la página

Ubicamos la etiqueta script justo antes de la etiqueta de cierre del body.

Así conseguimos que el navegador primero cargue y muestre el contenido del body y el CSS, y luego ejecuta el script.

Este método mejora el rendimiento visible (el usuario ve el contenido antes) y el script tiene acceso a todos los elementos HTML de la página; por eso es la práctica más recomendada.


Un archivo con la extensión .js es un archivo de texto plano que contiene solo código JavaScript.

Su propósito principal es separar la lógica (JS) de la estructura (HTML) y el estilo (CSS) de la página web. Además:

  • Mantiene el código limpio y fácil de mantener.
  • El mismo archivo .js puede ser enlazado y utilizado en múltiples páginas de HTML.
  • Los navegadores pueden almacenar en caché (temporalmente) los archivos .js externos, lo que acelera la carga de las páginas en visitas posteriores.

Se enlazan al HTML usando el atributo src en la etiqueta script.


El modo estricto es una característia de JavaScript introducida en EcmaScript 5.

Indica al código que debe ser ejecutaado en forma estricta.

Esto previene que se tomen ciertas acciones, se muestren más errores, y vuelve a este lenguaje más seguro.


    Los cambios que introduce el Modo Estricto son:

  • Hace algunos errores anteriormente silenciosos ahora arrojen excepciones;
  • Deshabilita ciertas características confusas o inconsistentes del lenguaje;
  • Facilita la optimización del código por parte de los motores JavaScript, pudiendo correr más rápido que el mismo código sin Modo Estricto;
  • Contribuye con producir código más seguro.

Para activar este modo, debemos colocar al inicio de nuestro archivo código o de nuestra función el siguiente código:

                
                

                "use strict"; 
                //o bien 
                'use strict';

                o en una función:

                function prueba (){
                'use strict';
                console.log("Probando estricto");
                }
