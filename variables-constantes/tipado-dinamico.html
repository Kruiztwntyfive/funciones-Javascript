JavaScript está tipado dinámicamente, lo que significa que no necesitamos declarar su tipo. Pero esto solo es posible porque identifica el tipo de variable que estamos declarando y la asigna, aunque no de forma explícita.
Ejemplo: 

                
                    const cliente = "Elizabeth";

No necesitamos decirle a JavaScript que la variable cliente es de tipo string porque automáticamente detecta que es de tipo string. 

                
                    
                    let primerNumero = 10;
                    let segundoNumero = 5;
                    let texto = "El resultado es: ";

El tipado fijo significa que el tipo de una variable se establece al momento de la declaración y no puede cambiar durante la vida de esa variable. La verificación de tipos se realiza antes de que el código se ejecute (en tiempo de transpilación).


TypeScript (TS) implementa esto al permitirte anotar tipos explícitamente a tus variables, parámetros de función y retornos. (vamos a tomar ejemplos de typescript únicamente para identificar el tipado fijo a las variables).


Características JavaScript - Dinámico TypeScript - Fíjo
Declaración No se especifica el tipo. El tipo se especifica con : después del nombre de la variable.
Código let cantidad = 10; let cantidad: number = 10;
Reasignación El tipo puede cambiar en cualquier momento. El tipo debe permanecer el mismo. 

// Ejemplo de Tipado Fijo   
let nombreUsuario: string = "Juliano";
nombreUsuario = "Elizabeth";  //Esto es correcto
nombreUsuario = 123; //Esto causaria error en la compilacion

// Tipado fijo en funciones
function saludar(nombre: string): void {
console.log(`Hola, ${nombre}.`);
}

saludar(nombreUsuario);

Beneficios del Tipado Fijo en TypeScript
La principal ventaja del tipado fijo en TypeScript es la detección temprana de errores:

  • Identificación de Errores: TypeScript detecta errores de tipo (como pasar una cadena donde se espera un número) mientras escribes o durante la compilación, antes de que el usuario vea el programa en funcionamiento.
  • Autocompletado Mejorado (IntelliSense): Cuando el editor conoce el tipo de una variable, puede sugerir automáticamente los métodos y propiedades disponibles para ese tipo, mejorando la productividad.
  • Código más Robusto y Mantenible: Al definir contratos de tipos claros para las funciones y variables, el código se vuelve más predecible y es más fácil de mantener y refactorizar, especialmente en proyectos grandes.



Scope Global (Ámbito Global)

El Scope Global es la región más externa de tu código.

  • Definición: Las variables y funciones declaradas fuera de cualquier función o bloque se encuentran en el Scope Global.
  • Accesibilidad: Pueden ser accedidas y modificadas desde cualquier parte del código, incluso dentro de funciones o bloques locales.
  • Advertencia: Es una mala práctica sobrecargar el scope global, ya que puede provocar conflictos de nombres (colisiones) con otras variables en tu código o librerías externas.

const nombreGlobal = "JavaScript"; // Ámbito Global

function mostrarNombre() {
    console.log(nombreGlobal); // Accede a la variable global
}

mostrarNombre(); // "JavaScript"
console.log(nombreGlobal); // "JavaScript"


Scope Local (Ámbito Local)

  • Definición: Las variables declaradas dentro de una función (utilizando var, let, o const) se encuentran en el Scope Local de esa función.
  • Accesibilidad: Solo pueden ser accedidas dentro de la función donde fueron definidas. No son visibles ni accesibles desde el exterior (el ámbito global) ni desde otras funciones.

function calcularSuma() {
  let numeroLocal = 10; // Ámbito Local de la función
  return numeroLocal + 5;
}

calcularSuma(); // Devuelve 15
// console.log(numeroLocal); // ❌ Error: numeroLocal is not defined


Scope de Bloque (Introducido por let y const)

  • Definición: Las variables declaradas con let o const dentro de un bloque de código (delimitado por llaves {}), como un if, for, o simplemente un par de llaves, están limitadas a ese bloque.
  • Importancia: var no respeta el scope de bloque, solo el scope de función. Usar let y const ayuda a prevenir el acceso inesperado a variables fuera del bloque donde fueron pensadas para usarse.

if (true) {
  const mensaje = "Hola, pertenece al bloque";
  var antiguo = "Soy VAR, Fui declarado en bloque";
  console.log(mensaje); 
}

// console.log(mensaje); // ❌ Error: mensaje is not defined (Está limitada al bloque if)
console.log(antiguo);   // ✅ Válido: "Soy VAR" (VAR ignora el scope de bloque)
